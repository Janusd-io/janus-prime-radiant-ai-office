# Skills will be written in input/activity/output ISO structure

## Decision

All skill documents will follow the ISO Figure-1 source-input-activity-output-receiver-control row structure so that every skill is automatically ISO-compliant in shape.

## Why

This collapses two problems into one: skills become documentation-grade ISO artefacts by construction, and the same template scales from micro-skills to macro evaluation processes. It also means any rendered diagram (graphic, Markdown, image) can be generated from the same table.
