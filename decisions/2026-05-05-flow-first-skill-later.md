# Document the evaluation flow first; convert to a Claude skill later

## Decision

The tool-evaluation pipeline will be documented as a flow (input/activity/output rows) before being formalised as a Claude skill, with the skill generated from the documented flow.

## Why

The flow is still evolving, so writing the skill prematurely would lock in an unstable shape. Documenting the flow first preserves optionality and lets Claude generate the skill once the flow stabilises, with the documentation itself becoming the institutional knowledge artefact.

## Evidence

> Speaker 1: To put a flow for now. Do the flow. You don't have to do a skill.
