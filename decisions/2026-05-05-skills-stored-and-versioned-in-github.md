# Skills are treated as source code and stored in GitHub for version control

## Decision

Skills are stored and versioned in GitHub so revisions are tracked, nobody can mutate them silently, and they can be promoted into Claude plugins for company-wide use.

## Why

Skills are software, not prose. They need version control, an approval workflow, and a library before they can be promoted to company-wide plugins. GitHub already gives the team revision history and access control.

## Evidence

> Speaker 1: We use GitHub for now because then GitHub's got a good version control. ... So we treat it like source code.
