# Build the ISO Figure-1 mapping bottom-up, one stage row at a time

## Decision

The AI Office will document the tool-evaluation process by filling in the ISO Figure-1 table one stage at a time (starting with Source + Input), rather than designing the full macro process top-down.

## Why

The process is complex (multiple gates, Linear stage sync, sub-processes for skill-building). Starting with Source and Input only keeps the team in sync, lets Simon ship something usable quickly, and the higher-stage rows can be added incrementally as understanding firms up.

## Evidence

> Speaker 1: It's easier to stack it back up than from top down... maybe we'll start with the sourcing input for now.
