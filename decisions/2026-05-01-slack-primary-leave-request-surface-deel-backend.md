# Slack stays the leave-request surface; Deel is the back-end record

## Decision

Employees will request and approve/reject leave through Slack (one company-wide requests hub); Deel will hold the authoritative leave-balance records and be updated via the same approval logic.

## Why

The company is consolidating employee requests in a single Slack hub to avoid scattering across multiple tools. Deel is coming online for HR records, so it will own balance/payroll truth while Slack provides the lightweight mobile-friendly request and approve interface mirroring the same logic.

## Evidence

> Theresa wong: I mean, for the leave request, if it's going to be working on deal, we link the same logic to deal. And the approve and disapprove will be from Slack.
