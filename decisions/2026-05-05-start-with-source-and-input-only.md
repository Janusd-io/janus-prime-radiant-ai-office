# Document only source and input first, defer activity/output

## Decision

The first deliverable from Simon will be the source + input rows of the evaluation framework; activity, output, receiver, control come in subsequent passes.

## Why

The pipeline is too large to document in one go and the skill itself is still evolving. Locking source and input first gives a stable foundation the rest of the company can also adopt for their own processes, and avoids over-specifying activity steps that will change as the flow matures.
