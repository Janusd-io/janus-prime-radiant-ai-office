**ATTRIBUTE REFERENCE**

All attributes used in this template fall into four categories. Each
attribute is defined once here and applies throughout the document
without repetition.

**A. Actor Attributes**

The following attributes identify who initiates, performs, routes,
decides, or receives. Their names differ by role but all draw from the
same registries:

|  |  |  |
|:---|:---|:---|
| **Attribute** | **Used In** | **Role** |
| **Initiator_ID** | Blocks 1, 3.5 | The party that originates or triggers an action |
| **Executor_ID** | Blocks 3.2, 3.3, 3.4 | The party that performs the action |
| **Routing_Target** | Block 3.1 | The party to whom the input is directed |
| **Decision_Maker** | Blocks 3.2, 3.4 | The party that makes the approval decision |
| **Receiver_ID** | Block 5 | The party that receives the output |

**Values (select one per attribute):** Person → User_ID (Corporate
Directory) \| Department → \[Company_ID\]\[Dept_ID\] (Organisation
Document Registry) \| System → System_ID (Software Registry) \| AI agent
→ AI_Model_ID (AI Models & Agents Registry)

**B. Location Attributes**

|  |  |  |
|:---|:---|:---|
| **Attribute** | **Used In** | **Meaning** |
| **Source_ID** | Blocks 1, 3, 5 | Unique code from the Data Sources Registry identifying the system or platform |
| **System_Code** | Block 3.1 | Same as Source_ID — used for the receiving system in Block 3.1 (Trigger Start Point) |
| **Record_Source_ID** | Blocks 3.1 – 3.5 | Same as Source_ID — specifies where a record or status entry is saved |
| **Sub_Location** | All blocks | A specific folder, channel, database, or path within the system identified by Source_ID / System_Code |

**C. Document & Record Attributes**

|  |  |  |
|:---|:---|:---|
| **Attribute** | **Used In** | **Meaning** |
| **Object_ID** | Blocks 2, 4 | Name/identifier of an input or output object following the corporate naming convention |
| **Record_Object_ID** | Blocks 3.1 – 3.5 | Same as Object_ID — used for records created or modified as evidence at each step |

**D. Required Registries**

|  |  |
|:---|:---|
| **Registry** | **Provides** |
| **Data Sources Registry** | Source_ID, Record_Source_ID, System_Code — see Appendix A.1 |
| **Software Registry** | System_ID (component of all actor attributes) — see Appendix A.2 |
| **AI Models & Agents Registry** | AI_Model_ID (component of all actor attributes) — see Appendix A.3 |
| **Organisation Document Registry** | Form_ID, Dept_ID, Object_ID, Record_Object_ID — see Appendix B |
| **Corporate Directory** | User_ID — internal employee account identifier |

| **BLOCK 2. INPUTS — Input Object Specifications and Formats** |

> **Entity Format**
>
> • Data: Text / Number / Link. Specify content requirements if
> applicable.
>
> • File (binary): specify required format (e.g., .pdf, .jpg).
> Prohibited formats across all processes: .exe, .sh, .bat.
>
> **Object_ID** — file or record name following the corporate naming
> convention:
>
> **Manual naming** (user-created files):
> **\[YYMMDD\]\_\[C\]\[D\]\[F\]\_\[UserID\]\_\[Ref\]**

|  |  |
|:---|:---|
| **Parameter** | **Description** |
| **\[YYMMDD\]** | Date of creation (e.g., 260603) |
| **\[C\]** | Company_ID — 1 digit (e.g., 1 = UAE, 2 = Singapore) — see Appendix B.1 Company Registry |
| **\[D\]** | Department_ID — 1 letter (e.g., F = Finance, H = HR, I = IMS, A = AI Native) — see Appendix B.2 |
| **\[F\]** | Form_ID — 1 digit from Organisation Document Registry (e.g., 1 = Contract, 2 = Invoice) |
| **\[UserID\]** | Employee identifier: personnel number, account ID, or EID |
| **\[Ref\]** | External reference: contract number, client code, or ticket ID. Use a sequential number if no external reference exists. |

> *Example: 260603_1F1_USR042_IBM-2024 — Finance contract created 3 June
> 2026 by employee USR042, referencing client IBM-2024.*
>
> **Auto-generated naming** (system-created files):
> **\[YYMMDD\]\_\[C\]\[D\]\[F\]\_\[SourceCode\]\_\[SystemID\]**

|  |  |
|:---|:---|
| **Parameter** | **Description** |
| **\[YYMMDD\]** | Date of creation |
| **\[C\]\[D\]\[F\]** | Company_ID, Department_ID, Form_ID — same as manual naming |
| **\[SourceCode\]** | Official Source_ID from the Data Sources Registry (e.g., 1P) |
| **\[SystemID\]** | Unique transaction or record ID assigned by the processing system (e.g., 168) |

> *Example: 260603_1F1_1P_168 — Finance contract auto-generated 3 June
> 2026 from Notion Production, transaction 168.*
>
> **External source naming** — preserve the original filename without
> modification.

**3.1 Trigger Start Point (the system that receives the input and the
event that starts the process)**

> **System_Code** — receiving system — from the Data Sources Registry —
> see Attribute Reference B
>
> **Sub_Location** — from Data Sources Registry — Appendix A.1
>
> **Routing_Target** — Person (User_ID Registry) \| Department

>
> **Description** — brief plain-text description of the triggering event
>
> *Example: "New AI tool request message posted in Slack

>
> *\[If applicable\]* **Record_Object_ID** — the document or record
> created at this trigger point — from Organisation Document Registry
> (Appendix B.2)
>
> *\[If applicable\]* **Record_Source_ID** — where the trigger record is
> saved — see Attribute Reference B

**3.2 Input Control (checking incoming inputs for compliance with
requirements)**

> **Source_ID** — from Data Sources Registry — Appendix A.1
>
> **Doc_Location** — where the control procedure document is stored;
> must specify: required characteristics, inspection methods, and
> acceptance criteria
>
> **Executor_ID** — Person (User_ID Registry) \| Department (\[C\]\[D\]
> — Org Doc Registry) \| System (Software Registry) \| AI agent (AI
> Models & Agents Registry)
>
> **Decision_Maker** — Person (User_ID Registry) \| Department
> (\[C\]\[D\] — Org Doc Registry) \| System (Software Registry) \| AI
> agent (AI Models & Agents Registry)
>
> **Status_Code**
>
> • Approved → continue process
>
> • Rejected → stop process; specify corrective action
>
> • Escalated → transfer to higher authority; specify escalation path
> and responsible party
>
> **Record_Object_ID** — exact artifact produced as evidence of this
> control step — from Organisation Document Registry (Appendix B.2)
>
> **Record_Source_ID** — from Data Sources Registry — Appendix A.1
>
> **Sub_Location** — from Data Sources Registry — Appendix A.1

**3.3 Scenario (step-by-step activities and in-process control points)**

> Each step follows this formula:

**Source_ID → Executor_ID → Action_Type → Object_Ref → Records**

|  |  |
|:---|:---|
| **Component** | **Content** |
| **Source_ID** | Where the action takes place — from Data Sources Registry (Appendix A.1) |
| **Executor_ID** | Who performs the action — Person (User_ID Registry) \| Department (\[C\]\[D\] — Org Doc Registry) \| System (Software Registry) \| AI agent (AI Models & Agents Registry) |
| **Action_Type** | What type of action — select from the Action_Type vocabulary below |
| **Object_Ref** | Reference to the procedure or specification document describing this step (with brief description), or plain-text description of the data entity acted upon |
| **Records** | Record_Object_ID + Record_Source_ID + Sub_Location — evidence of the step |
| **Sub_Process_Ref** | (optional) Name and Process_ID of the sub-process triggered by this step — used when this step initiates a separate defined process |

> **Action_Type vocabulary:**

|  |  |  |
|:---|:---|:---|
| **Code** | **Action** | **Description** |
| **CREATE** | Create | Generate a new document, record, or data object |
| **READ** | Read | Access or retrieve existing data without modification |
| **UPDATE** | Update | Modify an existing document or record |
| **DELETE** | Delete | Remove a document or record |
| **APPROVE** | Approve | Formally authorise or sign off on an object |
| **REJECT** | Reject | Formally decline or return an object for revision |
| **SEND** | Send | Transmit an object to another party or system |
| **RECEIVE** | Receive | Accept an incoming object from another party or system |
| **VALIDATE** | Validate | Check an object for compliance with defined criteria |
| **TRANSFORM** | Transform | Convert the format, structure, or content of data |
| **NOTIFY** | Notify | Send an alert or status communication |
| **ESCALATE** | Escalate | Transfer a decision to a higher authority |
| **ARCHIVE** | Archive | Move an object to long-term storage |

> *For in-process control points, apply the Input Control attributes
> (3.2) at the relevant step.*

> *\[If applicable\]* **Record_Source_ID** — from Data Sources Registry
