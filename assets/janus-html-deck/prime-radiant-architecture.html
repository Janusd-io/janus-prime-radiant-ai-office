Janus Prime Radiant · AI Office

How the AI Office instance of Prime Radiant moves information from raw signals → immutable sources → synthesised wiki → human and agent consumers. The substrate is a Git-backed Obsidian vault; the synthesis layer is software-agnostic and reachable from any combination of human (Obsidian), assistive agents (Claude Cowork, Claude Code), and the now-live per-person agent NanoClaw — running for one user via Slack DM with read-only access to its own clone of the vault.

Signals layer

Raw inputs the AIO instance perceives. Throw the net wide — under-collection is the failure mode. Each source type has its own ingest rule in CLAUDE.md §5.1.

Storage substrate

GitHub-backed Git repo per instance, cloned to each curator's machine. Migrated off Google Drive 2026-05-13. Obsidian Git plugin handles per-vault sync.

Synthesis layer

Atomic pages (entities, concepts, decisions, briefs, pulse, …) with frontmatter encoding four orthogonal edge types: entity, semantic, temporal, causal.

Interfaces

Obsidian (human), Cowork (primary agent), Claude Code (terminal-native git agent), and Nano — now live for one user (Michael, via Slack DM). Current posture: zero MCP connectors wired, vault clone read-only, no Prime Radiant skills bundled. Per-person scale-out pending Andrew's install.
