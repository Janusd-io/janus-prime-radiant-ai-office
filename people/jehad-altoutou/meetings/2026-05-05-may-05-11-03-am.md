## Summary

Working session between Michael Bruck, Jehad Altoutou, and Simon Tarskih to map the AI tool evaluation workflow onto the ISO 9001 Figure-1 schematic (source / input / activity / output / receiver / control) so the AI Office's tool-evaluation process can be IMS-documented. They walked through a worked example using a Slack tool-request as the Source and converged on: Slack message + URL is the Source; an auto-extracted form (requester, timestamp, URL, company, channel) is the Input; the evaluation skill (which embeds the AI policy) is the Activity; gates synchronise with Linear stages (Backlog / Evaluating / Sandbox / Approved or Rejected). They agreed skills are software and must be version-controlled in GitHub, that the AI policy is baked into each skill rather than passed as a separate input, and that finance / contracts are explicitly out of scope for the tool-evaluation process. They also opened a meta-thread on company-wide naming conventions (calendar invites, files, Slack channels) so downstream agents can classify and route automatically. Action items landed on Simon to migrate the evaluation table out of Word into a shared Google Doc and to draft the first two columns (Source, Input) of the table; the rest of the rows and the skill conversion are deferred.

## Findings

- Each row of the ISO Figure-1 table maps cleanly onto a Linear stage (Backlog -> Evaluating -> Sandbox -> Approved / Rejected), so the table doubles as the Linear pipeline spec. (stated by @jehad-altoutou)
- Skills are best framed as software (source code with version control, debugging, releases) rather than prose; this changes how they should be stored, reviewed, and promoted. (stated by @jehad-altoutou)
- Slack messages already carry enough structured metadata (user ID, timestamp, channel, attached URL, company-name string) to auto-populate a request form via JSON — Simon has a working sandbox demonstrating this. (stated by @simon-tarskih)
- Early evaluation gates (vendor ISO/SOC, training-data policy, Google/Slack integration support) can be fully automated; later stages (sandbox testing) are deliberately human-intensive. (stated by @jehad-altoutou)
- Pinecone has shipped an updated release adding agent-oriented features beyond pure vector RAG (TechCrunch piece shared in-meeting). (stated by @jehad-altoutou; confidence: medium)
- Each Slack workspace assigns a unique user ID per person per workspace; the same human has different IDs across Janus's different Slacks (Genesis vs 71 Capital), which has implications for cross-workspace identity reconciliation. (stated by @jehad-altoutou)
- Industry trend: organisations are migrating from Microsoft Teams to Slack despite the migration cost, on the basis of Slack's automation and agent-integration potential. (stated by @jehad-altoutou; confidence: low)
