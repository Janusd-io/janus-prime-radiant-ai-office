01 · Track A — substrate migration

Prime Radiant lives on Git now.

Drive's streaming-mount placeholder layer was breaking Cowork's recursive mount walk and blocking new-contributor onboarding. We diagnosed it, decided, migrated, and shipped the runbook in a single week — Mon 11 May to Wed 13 May.

Migration sequence (13 May)
  • Repo Janusd-io/janus-prime-radiant-ai-office created (private). AI-Office team + Michael + Jehad as collaborators.
  • Vault materialised from Drive, moved via Finder to ~/janus/prime-radiant/ai-office/.
  • .gitignore + git init --initial-branch=main + initial commit + push.
  • Obsidian Git plugin configured: auto-pull on startup, 5-min auto-commit-and-sync, pull-before-push.
  • Cowork project repointed at the new local path; mount verified clean.
What this buys us
  • Reads see real files. No first-walk warmup quirk; what Obsidian sees is what Claude sees.
  • Identity decoupled. GitHub Teams replaces cross-Workspace Drive identity — fixes Andrew's onboarding failure mode at the substrate layer.
  • Federation aligns with the substrate. Sibling clones in ~/janus/prime-radiant/ map one-to-one to the mesh-federation design intent.
  • Git history is the audit trail. Complements the append-only log.md.
Two GitHub orgs locked in
Janusd-io for AIO + the template repo. Janusd-com for every other department instance (PM, IT, Marketing, ISO, HR, Finance, Office-of-CEO, Engineering, Training). Keeps AIO experimentation isolated from company-operational vaults while reusing the same Prime Radiant template + tooling.
Curator runbook extracted
New processes/prime-radiant-instance-setup.md with embedded bash script automates steps 1, 3, 4, 5, 6 of the migration sequence. GUI-only steps (collaborators, branch protection, Obsidian, Cowork) walked through in prose. Expected scaffold time: ~2h unscripted → ≤10 min scripted.
