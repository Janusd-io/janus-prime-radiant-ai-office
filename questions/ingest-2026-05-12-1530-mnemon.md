## What Mnemon is, in one paragraph

Single-binary Go program that gives LLM-CLI agents (Claude Code, [[openclaw]], NanoClaw, OpenCode, Gemini CLI) persistent cross-session memory backed by a four-graph store (temporal/entity/causal/semantic) plus optional vector via local Ollama. The pattern: the binary handles deterministic storage; the host LLM (running on the user's existing Claude/OpenAI/etc. subscription) makes all judgment calls — what to remember, how to link, when to forget. Installs through a markdown harness (`SKILL.md` + `GUIDELINE.md` + four lifecycle hooks: Prime / Remind / Nudge / Compact). Vision: one shared `~/.mnemon` pool across all of a user's CLI agents.

## Why this is the borderline case

**Arguments for creating a vendor page:**
- Mnemon proposes a genuinely new pattern axis (LLM-role taxonomy) that's now baked into [[agent-memory]] and [[agent-memory-2026-q2]]. Future ingests that touch agent memory will likely cite it.
- It's the closest external system to the Prime Radiant discipline (user-owned files, structured graphs, LLM-as-supervisor) — worth a durable entity record so the pattern comparison has a stable target.
- It explicitly targets [[claude]] Code, [[openclaw]] and NanoClaw — three vendors already in the wiki — so cross-references would have natural homes.

**Arguments against (yet):**
- Single-source ingest. Per the wiki's discipline, vendor pages survive without bloat (CLAUDE.md §6), but a vendor known only from a README is thin evidence.
- No adoption signal in this batch: no star history, no benchmark, no production deployment story, no third-party validation of the four-graph architecture claims.
- The MAGMA + RLM + Graph-LLM paper grounding is asserted but unverified in this ingest.
- Janus has no live evaluation of Mnemon; the [[ai-tool-evaluation]] gate framework hasn't been pointed at it.
- Comparable open-source memory projects (Mem0, Letta, claude-mem, MemPalace) currently exist in the wiki only via pulse/brief/concept references — no vendor entity for any of them. Creating one for Mnemon without elevating the peer set introduces taxonomy asymmetry.

## Alternative interpretations / options

1. **Create the vendor page now.** Confidence: low. Single-source, README-only, untested. Sets a relaxed bar for vendor-page creation — future README-only ingests would expect similar treatment.
2. **Defer until a second source surfaces.** Treat the pulse entry + brief vendor-signals row as the durable record for now. Promote to vendor page when (a) a second source mentions Mnemon, OR (b) someone at Janus actually evaluates/uses it, OR (c) the pattern axis it introduced needs a more authoritative home. *(Recommended default — matches the Q2 OpenAI deferral precedent in the `2026-05-07 20:00` ingest log entry: "no new entity creation; OpenAI as a vendor page is borderline — defer until 2nd source warrants it".)*

Higher overall cost but resolves the taxonomy asymmetry.
