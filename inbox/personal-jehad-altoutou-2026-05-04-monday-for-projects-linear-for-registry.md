# Split tool tracking from project tracking: Linear for AI Registry, Monday for projects

## Decision

The AI Registry stays in Linear (system of record for tools); active projects and granular tasks live in Monday.com because Linear was not tracking project work well.

## Why

Linear fits CRM-like pipeline stages (Backlog/Evaluating/Sandbox) for tools; Monday's project-manager granularity is better for sub-items, assignees and progress tracking on Jehad's project workstreams. Notion remains the meeting log.

## Evidence

> Michael Bruck: we're actually using for the registry linear. Because it fits in very well. But for all the projects ... monday.com.
