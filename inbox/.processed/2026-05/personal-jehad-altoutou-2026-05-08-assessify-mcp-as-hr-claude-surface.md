# Assessify exposes recruitment pipeline to Claude via MCP

## Decision

The recruitment pipeline (JD creation, candidate intake, rubric scoring) is driven through an MCP connector to Assessify so HR can operate everything from Claude rather than the platform UI.

## Why

Jehad has built the MCP server already; HR users can create JDs, pull candidates, and run scoring from inside Claude. This lowers the operational barrier for HR and keeps all interactions captured for the wiki.
