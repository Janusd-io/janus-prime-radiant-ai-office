Three constraints shape every choice we make

The marketing function is being stood up under conditions that rule out most institutional stack designs.

Team size

Marketing is a team of one today, with engineering support from the AI Office and infrastructure support from IT. Additional hires through 2026 are in scope but subject to approval — likely 2–3 for marketing and martech combined, and hiring takes time. Singapore ships with the team we have right now.

Speed to first launch

Singapore campaign launches in roughly ten days. Thought leadership lunch July 8/9. Decisions made now have to ship on this timeline.

Multi-region rollout

Singapore is first. UK follows. Additional regions through 2026 and 2027. Whatever pattern we set becomes the template that compounds across every region.

The question isn't "what's the best stack?" — it's "what's the best stack we can actually run?"

What's changed structurally in the last 18 months

Two specific shifts have moved the breakeven point for running a multi-tool stack with a lean team.

MCP became the standard

Anthropic open-sourced the Model Context Protocol in late 2024. By mid-2026, native MCP servers ship from Salesforce, HubSpot, Attio, Vercel, Cloudflare, Sanity, Cosmic, Adobe, and several hundred others.

It's effectively USB-C for AI tool connectivity. Claude Code can operate any of these without custom integration work.

Claude Code matured

Vercel reports 30% of all deployments on its platform are now initiated by coding agents — a tenfold increase in six months. Claude Code accounts for 75% of those.

Local proof point: Jehad refactored janusd.com from static HTML to TypeScript/React in about an hour last week.

Together, these collapse the operating cost of running a multi-tool stack — for tools selected on this basis.

Two genuinely viable paths

Same business outcomes — leads, nurture, content velocity, multi-region scale — achievable two different ways.

Path A

Traditional best-of-breed

Salesforce + Adobe + Marketo or HubSpot Marketing Hub + supporting analytics and content tools.

Capability: Very high. Deepest tools in each category.
Ops floor: Very high. Even a stripped-down version requires 4–6 dedicated marketing-tech specialists.
Right for: Enterprises with the team to operate it.
Path B · Recommended

Agent-native composable

Cosmic + Attio + Resend + Vercel + Cloudflare, with Claude Code as the operational substrate.

Capability: High. Meets all 2026/2027 marketing objectives.
Ops floor: Low. Runnable by the team in place today plus the additions we make this year.
Right for: Our stage, our team, our timeline.

Path A isn't wrong — it's optimized for an operating model we don't have today.

The framework for every tool decision

Any tool we add gets evaluated against three questions before it enters our existing AI Tool Evaluation gates.

1

Composability

Does the data model map cleanly to adjacent tools, or do we need custom sync layers?

Why it matters

Prevents a 'best-in-class' tool collection that doesn't actually integrate.

2

Agent operability

Does it ship a real CLI, a robust API, and a native MCP server, so Claude Code can drive it?

Why it matters

Prevents stack expansion that outruns our operational capacity.

3

Reversibility

If it doesn't work out in 12–18 months, can we migrate off in a reasonable timeframe?

Why it matters

Prevents vendor lock-in that undermines future flexibility.

3/3 = preferred default · 2/3 = viable with documented trade-off · 1/3 = generally reject

The recommended stack

Each layer chosen because it composes with the others, is agent-operable, and stays replaceable.

Layer Recommended Why
Hosting & edge Vercel + Cloudflare Full CLIs + official MCP servers. Multi-region routing and regional IP resolution built-in.
Application Next.js (single codebase) Already built. One repo serves all regional domains.
CMS Cosmic Native MCP + Agent Skills + REST API. Content stays as portable JSON.
CRM Attio Designed agent-first. Data model and MCP built together.
Email (transactional) Resend Already in place. Strong API and CLI.
Email (marketing) Resend Broadcasts or Mailchimp Decided after Singapore launch based on cadence needs.
Privacy & cookie consent Cookiebot / Iubenda / similar Multi-jurisdiction compliance (GDPR, PDPA, CCPA) handled by a third-party service. Inserted via script tag; no custom code to maintain per region.
Code & deployment Claude Code + GitHub The operational substrate that ties the rest together.

Total run rate at full capacity: $1,200–$1,800/mo in licensing, plus Claude Code subscriptions. The meaningful cost difference vs. Path A is the FTEs we're not adding — not the licensing.

What this means for marketing specifically

The agent-native stack isn't a constraint on what marketing can do — it's the multiplier that makes a team of one (plus the additions through the year) viable.

Content doesn't queue behind engineering

Marketing describes what's needed in plain language. Claude Code, working through Cosmic, Resend, and Attio, executes it. Campaign landing pages, email sequences, content updates — all unblocked from the dev queue.

New regions deploy in days, not months

The template built for Singapore is reusable. UK launch becomes a clone exercise — new bucket, new domain, same code. Every region after that follows the same pattern, measured in days from the Singapore baseline.

Time stays focused on the high-leverage work

Strategy, customer conversations, brand positioning, campaign narrative — the parts of CMO work that only a human can do. Tool operation moves to agents. Effectively, marketing has the leverage of a six-to-eight person team.

AI Office handles the software engineering layer. IT handles infrastructure and DevOps. Marketing operates above both, through agents.
